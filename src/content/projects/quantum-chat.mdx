# Quantum Chat

Quantum Chat redefines speed. Built for communities where every millisecond counts, it serves as a proof-of-concept for high-frequency data transmission over standard WebSockets.

## The Challenge

Standard chat apps suffer from latency spikes. Quantum solves this by using a custom binary protocol over WebSockets, handled by a highly concurrent Rust backend.

## Performance Metrics

- **Latency**: < 5ms average globally.
- **Throughput**: 100k messages/second per node.
- **Uptime**: 99.99%.

## Code Snippet (Rust Backend)

```rust
use actix_web::{web, App, HttpServer};

async fn index() -> String {
    "Hello quantum world!".to_string()
}

#[actix_web::main]
async fn main() -> std::io::Result<()> {
    HttpServer::new(|| App::new().route("/", web::get().to(index)))
        .bind("127.0.0.1:8080")?
        .run()
        .await
}
```
