## The Part Where It Does Real Security

Nobody:\
Enterprise WAF vendors: "Have you completed our 14-step onboarding wizard?"

Here's what's actually running under the hood — and yes, it cares about your OWASP Top 10 more than the SaaS tool that sent you a welcome email:

- **Regex Pattern Matching:** Catches SQLi (`union`, `select`, `drop`, `sleep`), XSS (`<script>`, `onerror=`), and path traversal (`../`) before your backend ever sees a request.
- **Token-Bucket Rate Limiting:** Per-IP, per-path, configurable globally. Your `/api/login` will not survive 500 requests per second anymore. Good.
- **Body Inspection:** POST request bodies get the full treatment. Your text fields cannot be injection vectors.
- **Fail-Safe Mode:** Rule engine crashes — choose "Fail-Open" for uptime priority or "Block" when security actually matters.

The `AtomicEngine` handles **hot reload**. Edit `rules.yaml` live, updates apply in **2 seconds** without dropping any active connections. Realtime security config changes on a hobby budget. Completely unhinged in the best way.

```go
// The Rule interface: Simple, powerful, and modular.
type Rule interface {
    Name() string
    Evaluate(r *http.Request) (blocked bool, reason string)
}
```
