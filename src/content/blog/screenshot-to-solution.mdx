# From Screenshot to Solution

> "The hardest part of fixing a bug isn't the code—it's describing it."

Bug reporting in 2026. Still a game of telephone. Still completely broken.

"It's not working." — the full Jira ticket description sitting in your queue right now.

We live in a timeline where AI writes production code, yet the standard bug report is: screenshot cropped to a vague red box, pasted into Slack, followed by four words and the prayer that the developer receiving it is also psychic. By the time that reaches an AI coding agent, the context has fully decayed. The agent writes the wrong fix. You spend 20 minutes explaining what was broken. Classic.

Enter **AI Feedbacks** — which exists specifically because someone got tired of receiving "it doesn't work" as actionable development context.

![AI Feedbacks Logo](/ai-feedbacks-logo.png)

## The Problem: Context Decay (AKA Your Entire Morning)

When a bug fires, a bunch of incredibly useful information exists for exactly a few seconds: the DOM state, the console errors, the failed network requests, the exact UI configuration at the moment of failure. Then the user closes the tab, writes "the button did something weird," attaches a screenshot of the wrong screen, and submits.

By the time that ticket reaches an agent like Cursor or Windsurf, the prompt is so thin it's basically a riddle. Developers end up spending 20 minutes reconstructing what happened before they can even start fixing it. Coding agents hallucinate because they have no context. Everyone loses.

The technical term is "context decay." The honest term is "why does this keep happening."

## The Solution: Give Your Agent Enough to Actually Work With

AI Feedbacks captures the full crash scene at the moment it happens — screenshot, DOM context, console errors, failed network requests — and synthesizes it into a structured, agent-ready prompt. No manual log-copying. No vague description. Just: here is exactly what broke, in the format your agent needs to fix it in one pass.

**Google Gemini 3 Flash Preview** does the synthesis:

```ts
// The AI processing pipeline
const result = streamText({
  model: google('gemini-3-flash-preview'),
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: `Analyze this bug: ${description}` },
        { type: 'image', image: screenshotBuffer },
      ],
    },
  ],
})
```

Gemini identifies the broken component, cross-references the network logs, and returns a structured prompt. The kind that works on the first agent pass instead of requiring five clarification messages. Wild concept.

### How the Chrome Extension Captures Context

The extension runs silently in the background and on crop+submit:

- Grabs the exact area of the screen you flagged
- Captures all `console.error` and `console.warn` from the session
- Packages failed network requests (4xx/5xx) from the last 60 seconds
- Sends the whole thing to the backend without you doing anything

The developer receives a prompt. Not a screenshot. A prompt.

## The Dashboard: Because You Also Need Last Tuesday's Bug

The web dashboard is built with **natural language semantic search** because nobody has time to remember the exact words they used in a ticket from two weeks ago.

Search for:

- _"that login error"_
- _"the white flash on mobile"_
- _"whatever was wrong with the chart last week"_

And it finds it. Not because you tagged it correctly. Because you described it like a human.

The UI is monochrome, high-contrast, and deliberately boring-looking. The tool is the content. No decorative gradients competing for your attention while you're trying to ship.

## Who This is Pointedly For

If you or your team uses Cursor, Windsurf, GitHub Copilot, or literally any agentic tool for development — you already know the dirty secret: the quality of the output is entirely constrained by the quality of the prompt. AI Feedbacks automates the hardest part of writing a good bug prompt: capturing and structuring the context at the exact moment it exists.

Feed your agents better. Get first-pass fixes more often. It's not rocket science, it's just better tooling.
