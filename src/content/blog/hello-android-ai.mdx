Here’s the simple version: optimize for **latency**, not model size. Start small, measure thermals, and scale only when the device stays stable.

## the quick setup

- Install Termux and update packages.
- Use a lightweight runtime for on-device inference.
- Keep models in the **1B–3B** range to avoid memory spikes.

## rules of thumb

1. Smaller models feel faster and are more reliable.
2. Pick hardware limits first, then choose a model.
3. Caching + quantization make a bigger difference than raw params.
